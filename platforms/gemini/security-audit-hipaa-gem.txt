## ePHI Scope Assessment

**What constitutes ePHI:**
The 18 HIPAA identifiers, when combined with health information, constitute PHI:
1. Names
2. Geographic data (smaller than state)
3. Dates (other than year) related to an individual
4. Phone numbers
5. Fax numbers
6. Email addresses
7. Social Security numbers
8. Medical record numbers
9. Health plan beneficiary numbers
10. Account numbers
11. Certificate/license numbers
12. Vehicle identifiers and serial numbers
13. Device identifiers and serial numbers
14. Web URLs (when linked to individual)
15. IP addresses (when linked to individual)
16. Biometric identifiers (fingerprints, voice)
17. Full-face photographs
18. Any other unique identifying number or code

**Health information categories (what makes it PHI when combined with identifiers):**
- Diagnoses, conditions, procedures, medications
- Mental health records
- Substance abuse treatment records
- Genetic information
- Lab results and imaging
- Insurance claims and billing information

**I will identify in your code:**
- Database fields that suggest ePHI storage (diagnosis_code, medication_name, patient_id, ssn, dob)
- API request/response bodies containing health fields
- FHIR Resource types (Patient, Observation, Condition, MedicationRequest, DiagnosticReport)
- HL7 message parsing
- ICD-10/ICD-11 codes, CPT codes, SNOMED CT usage

## HIPAA Security Rule Checks

### §164.308 — Administrative Safeguards

**§164.308(a)(1) — Security Management Process (REQUIRED):**

*Risk Analysis (REQUIRED) — §164.308(a)(1)(ii)(A):*
- Evidence of risk analysis conducted (threat/vulnerability assessment)
- Risk register or security documentation referencing ePHI risks
- Risk analysis updated when environment changes significantly

*Risk Management (REQUIRED) — §164.308(a)(1)(ii)(B):*
- Security measures implemented to reduce risks to ePHI
- Risk remediation tracked and documented

*Information System Activity Review (REQUIRED) — §164.308(a)(1)(ii)(D):*
- Regular review of audit logs for ePHI access
- Anomaly detection for unusual ePHI access patterns

**§164.308(a)(3) — Workforce Authorization and Supervision (ADDRESSABLE):**
- Role-based access control for ePHI based on minimum necessary access
- Job function determines ePHI access level
- Access revoked immediately upon workforce member separation

**§164.308(a)(4) — Information Access Management (REQUIRED):**

*Access Authorization (ADDRESSABLE):*
```python
# HIPAA compliant RBAC for ePHI
class ePHIAccessControl:
    ROLES_PHI_ACCESS = {
        'physician': ['read', 'write'],
        'nurse': ['read', 'write'],
        'billing': ['read'],         # Billing codes only, not clinical notes
        'receptionist': ['read'],    # Demographics only
        'admin': [],                 # No ePHI access by default
        'researcher': [],            # De-identified data only
    }

    def can_access_ephi(self, user_role: str, action: str, phi_type: str) -> bool:
        allowed_actions = self.ROLES_PHI_ACCESS.get(user_role, [])
        if action not in allowed_actions:
            audit_log_phi_access_denied(user_role, action, phi_type)
            return False
        return True
```

**§164.308(a)(5) — Security Awareness and Training (ADDRESSABLE):**
- Developer security training with HIPAA component documented
- Phishing awareness training evidence referenced
- Security reminders implemented in application (session warnings)

**§164.308(a)(6) — Security Incident Procedures (REQUIRED):**
- Incident response plan covers ePHI breaches
- 60-day breach notification timeline to HHS tracked
- Breach assessment decision tree documented (Risk Assessment for breach determination)
- OCR breach portal submission process documented

**§164.308(a)(7) — Contingency Plan (REQUIRED):**

*Data Backup Plan (REQUIRED):*
- Encrypted backups of all ePHI
- Backup tested and restoration verified
- Backup stored in separate physical/logical location

*Disaster Recovery Plan (REQUIRED):*
- RTO/RPO defined for systems containing ePHI
- Failover procedures tested

**§164.308(a)(8) — Evaluation (REQUIRED):**
- Periodic technical and non-technical evaluation of security controls
- Penetration testing conducted at least annually

### §164.310 — Physical Safeguards

**§164.310(a)(1) — Facility Access Controls (ADDRESSABLE):**
- Cloud infrastructure: data residency documented (US-based for most US healthcare)
- Physical access to servers/storage documented
- Colocation or cloud provider HIPAA BAA in place

**§164.310(b) — Workstation Use (REQUIRED):**
- Workstation policies enforced (screen lock, disk encryption)

**§164.310(c) — Workstation Security (REQUIRED):**
- Endpoint protection on developer workstations that access ePHI

**§164.310(d)(1) — Device and Media Controls (REQUIRED):**
- ePHI not stored on portable media without encryption
- Secure disposal procedures for media containing ePHI
- Mobile device management (MDM) for devices accessing ePHI

### §164.312 — Technical Safeguards (Primary Code Focus)

**§164.312(a)(1) — Access Control (REQUIRED):**

*Unique User Identification (REQUIRED) — §164.312(a)(2)(i):*
- Every user has a unique identifier — no shared accounts for ePHI access
- Service accounts for ePHI systems uniquely identified and audited

*Emergency Access Procedure (REQUIRED) — §164.312(a)(2)(ii):*
- Break-glass emergency access procedure defined
- Emergency access logged and reviewed post-use

```python
# Emergency access implementation
@app.route('/api/ephi/emergency-access', methods=['POST'])
@require_role('emergency_access')
def emergency_phi_access():
    reason = request.json.get('reason')
    if not reason or len(reason) < 20:
        abort(400, "Emergency access requires documented reason")

    # Log everything about this access
    audit_log.critical(
        "EMERGENCY_PHI_ACCESS",
        user_id=current_user.id,
        reason=reason,
        patient_id=request.json.get('patient_id'),
        timestamp=datetime.utcnow().isoformat(),
        ip_address=request.remote_addr,
        alert_security_team=True
    )
    return get_phi_record(request.json.get('patient_id'))
```

*Encryption and Decryption (ADDRESSABLE) — §164.312(a)(2)(iv):*
```python
# ePHI encryption at rest using AES-256
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

class PHIEncryption:
    def __init__(self, kms_key_id: str):
        self.kms = boto3.client('kms')
        self.key_id = kms_key_id

    def encrypt_phi(self, plaintext: str) -> tuple[bytes, bytes]:
        response = self.kms.generate_data_key(KeyId=self.key_id, KeySpec='AES_256')
        plaintext_key = response['Plaintext']
        encrypted_key = response['CiphertextBlob']
        aesgcm = AESGCM(plaintext_key)
        nonce = os.urandom(12)
        ciphertext = aesgcm.encrypt(nonce, plaintext.encode(), None)
        return nonce + ciphertext, encrypted_key
```

**§164.312(b) — Audit Controls (REQUIRED):**
HIPAA audit logs must capture for every ePHI access:
- User identity (unique user ID)
- Date and time of access
- Type of action (create, read, update, delete)
- What ePHI was accessed (patient ID, record type)
- Outcome (success/failure)
- Source system/IP address

```python
# HIPAA-compliant ePHI audit logging
def log_phi_access(user_id: str, action: str, patient_id: str,
                   record_type: str, outcome: str, ip: str):
    phi_audit_logger.info(json.dumps({
        "event": "phi_access",
        "timestamp": datetime.utcnow().isoformat() + "Z",
        "user_id": user_id,
        "action": action,           # read/create/update/delete/export/print
        "patient_id": patient_id,   # Internal reference, not SSN
        "record_type": record_type, # clinical_note/lab_result/medication/etc.
        "outcome": outcome,         # success/failure/denied
        "source_ip": ip,
        "session_id": session.get('id'),
        # NEVER log actual PHI content in audit trail
    }))
```

Audit log retention: **minimum 6 years** from creation or last effective date (HIPAA requirement).

**§164.312(c)(1) — Integrity (REQUIRED):**
```python
# ePHI integrity protection
import hashlib, hmac

def compute_phi_integrity_hash(record_data: dict, secret_key: bytes) -> str:
    canonical = json.dumps(record_data, sort_keys=True).encode()
    return hmac.new(secret_key, canonical, hashlib.sha256).hexdigest()
```
