## CIS Controls v8 Checks (Implementation Group 1-2)

### IG1 — Basic Cyber Hygiene (Controls 1-8)

**CIS Control 1 — Inventory and Control of Enterprise Assets:**
- SBOM generated for containers and applications
- Asset inventory maintained (container registry with tagged images)

**CIS Control 2 — Inventory and Control of Software Assets:**
- All software components inventoried (package-lock.json, go.sum, SBOM)
- Unauthorized software blocked (allowlist-only container registries)

**CIS Control 4 — Secure Configuration of Enterprise Assets and Software:**
- CIS Docker Benchmark applied to all container builds
- CIS Kubernetes Benchmark applied to all cluster configs
- CIS Cloud Benchmarks applied to cloud accounts

**CIS Control 5 — Account Management:**
- No default credentials in use
- Service accounts follow least privilege
- Root account usage prohibited

**CIS Control 6 — Access Control Management:**
- RBAC enforced in Kubernetes
- IAM least privilege in cloud
- MFA on all privileged accounts

**CIS Control 7 — Continuous Vulnerability Management:**
- Container image scanning on push and periodically
- OS package CVE scanning
- Dependency scanning with SLA enforcement

**CIS Control 8 — Audit Log Management:**
- Kubernetes audit logging enabled
- Container runtime logs captured
- Log retention minimum 90 days

## CIS Docker Benchmark v1.6 Checks

### Section 1 — Host Configuration

**CIS 1.1 — Ensure the container host has been hardened:**
- Docker host runs a minimal OS (Alpine, Flatcar, Container Optimized OS)
- Unnecessary packages removed from Docker host

**CIS 1.2 — Ensure Docker is updated:**
- Docker version pinned and current (no critical CVEs in Docker daemon)

**CIS 1.6 — Ensure auditing is configured for Docker daemon:**
```bash
# Verify auditd rules for Docker
auditctl -l | grep docker
# Should have rules for:
# /usr/bin/dockerd
# /var/lib/docker
# /etc/docker
```

### Section 2 — Docker Daemon Configuration

**CIS 2.1 — Ensure network traffic is restricted between containers on the default bridge network:**
```json
// /etc/docker/daemon.json
{
  "icc": false,  // CIS 2.1: Inter-container communication disabled by default
  "live-restore": true,
  "userland-proxy": false,
  "no-new-privileges": true,
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

**CIS 2.2 — Ensure logging level is set to 'info':**
- Docker daemon log level set to info (not debug in production)

**CIS 2.14 — Ensure Usernamespace support is enabled (L2):**
- User namespace remapping configured in daemon.json

### Section 4 — Container Images and Build File

**CIS 4.1 — Ensure that a user for the container has been created (CRITICAL — Level 1):**
```dockerfile
# VIOLATION: Running as root
FROM ubuntu:22.04
# No USER instruction — runs as root by default

# COMPLIANT: Non-root user
FROM ubuntu:22.04
RUN groupadd -r appuser && useradd -r -g appuser appuser
WORKDIR /app
COPY --chown=appuser:appuser . .
USER appuser  # CIS 4.1 compliant
```

**CIS 4.2 — Ensure that containers use only trusted base images (Level 1):**
```dockerfile
# VIOLATION: Unverified base image
FROM ubuntu:latest      # Mutable tag, unverified source
FROM some-user/myimage  # Unverified third-party image

# COMPLIANT: Official image, pinned digest
FROM ubuntu:22.04@sha256:a1b2c3...  # CIS 4.2 compliant
FROM node:20-alpine@sha256:d4e5f6...
```

**CIS 4.3 — Ensure that unnecessary packages are not installed in the container (Level 1):**
```dockerfile
# VIOLATION: Installing debugging tools in production image
RUN apt-get install -y curl wget netcat nmap vim

# COMPLIANT: Minimal packages only
RUN apt-get install -y --no-install-recommends \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*
```

**CIS 4.5 — Ensure Content trust for Docker is enabled (Level 2):**
- DOCKER_CONTENT_TRUST=1 set in CI/CD environment
- Images signed with Cosign/Notary before deployment

**CIS 4.6 — Ensure that HEALTHCHECK instructions have been added to container images (Level 1):**
```dockerfile
# VIOLATION: No health check
FROM node:20-alpine
# ... build steps ...
CMD ["node", "server.js"]
# No HEALTHCHECK instruction

# COMPLIANT:
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1
```

**CIS 4.7 — Ensure update instructions are not used alone in the Dockerfile (Level 1):**
```dockerfile
# VIOLATION: Update in separate layer risks stale cache
RUN apt-get update
RUN apt-get install -y nginx  # May use cached, outdated apt index

# COMPLIANT: Combine in single layer
RUN apt-get update && apt-get install -y --no-install-recommends nginx \
    && rm -rf /var/lib/apt/lists/*
```

**CIS 4.9 — Ensure that COPY is used instead of ADD in Dockerfiles (Level 1):**
```dockerfile
# VIOLATION: ADD with URL — downloads from internet at build time
ADD https://example.com/myfile.tar.gz /app/

# COMPLIANT: Use COPY for local files
COPY myfile.tar.gz /app/
RUN tar -xzf /app/myfile.tar.gz
```

**CIS 4.10 — Ensure secrets are not stored in Dockerfiles (Level 1):**
```dockerfile
# CRITICAL VIOLATION: Secrets in Dockerfile
ENV AWS_SECRET_ACCESS_KEY=mysecretkey
RUN curl -H "Authorization: Bearer hardcoded_token" https://api.example.com

# COMPLIANT: Secrets via build secrets (BuildKit)
RUN --mount=type=secret,id=api_token \
    curl -H "Authorization: Bearer $(cat /run/secrets/api_token)" https://api.example.com
```

### Section 5 — Container Runtime

**CIS 5.1 — Ensure that, if applicable, an AppArmor profile is enabled (Level 1):**
```bash
# docker run with AppArmor profile
docker run --security-opt apparmor=docker-default myimage
```

**CIS 5.2 — Ensure that, if applicable, SELinux security options are set (Level 2):**
```bash
docker run --security-opt label=type:svirt_lxc_net_t myimage
```

**CIS 5.4 — Ensure that privileged containers are not used (Level 1 — CRITICAL):**
```yaml
# docker-compose.yml VIOLATION
services:
  app:
    image: myapp
    privileged: true  # CIS 5.4 CRITICAL violation — full host access

# COMPLIANT: No privileged mode
services:
  app:
    image: myapp
    security_opt:
      - no-new-privileges:true
    read_only: true
    cap_drop:
      - ALL
    cap_add:
      - NET_BIND_SERVICE  # Only if needed
```

**CIS 5.7 — Ensure privileged ports are not mapped within containers (Level 1):**
```yaml
# VIOLATION: Mapping privileged port (< 1024)
ports:
  - "80:80"  # Binding to port 80 inside container as root

# COMPLIANT: Use unprivileged port inside, map with reverse proxy
ports:
  - "8080:8080"  # Non-privileged port inside container
```

**CIS 5.12 — Ensure the container's root filesystem is mounted as read only (Level 1):**
```yaml
# docker-compose.yml COMPLIANT
services:
  app:
    image: myapp
    read_only: true
    tmpfs:
      - /tmp:noexec,nosuid,size=64m  # Writable tmp if needed
      - /var/run:noexec,nosuid,size=10m
```

**CIS 5.15 — Ensure that the host's process namespace is not shared (Level 1):**
```yaml
# VIOLATION
pid: host  # Shares host PID namespace

# COMPLIANT: Remove pid: host (default is isolated)
```

**CIS 5.31 — Ensure that the Docker socket is not mounted inside any containers (Level 1):**
```yaml
# CRITICAL VIOLATION: Docker socket mount = container escape
volumes:
  - /var/run/docker.sock:/var/run/docker.sock

# COMPLIANT: Remove this volume mount entirely
# Use Docker-in-Docker (dind) or Kaniko for CI builds instead
```

## CIS Kubernetes Benchmark v1.8 Checks

### Section 4 — Worker Node Security Configuration

**CIS K8s 4.2.6 — Ensure that the --protect-kernel-defaults argument is set to true:**
```yaml
# kubelet configuration
protectKernelDefaults: true
```

### Section 5 — Kubernetes Policies

**CIS 5.1.1 — Ensure that the cluster-admin role is only used where required (Level 1):**
```yaml
# VIOLATION: ClusterRoleBinding with cluster-admin for a service account
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: bad-binding
subjects:
- kind: ServiceAccount
  name: myapp
  namespace: default
roleRef:
  kind: ClusterRole
  name: cluster-admin  # CIS 5.1.1 VIOLATION
  apiGroup: rbac.authorization.k8s.io

# COMPLIANT: Namespace-scoped Role with minimal permissions
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: production
  name: myapp-role
rules:
- apiGroups: [""]
  resources: ["configmaps"]
  verbs: ["get", "list"]  # Read-only on configmaps only
```

**CIS 5.2.2 — Minimize the admission of containers wishing to share the host process ID namespace (Level 1):**
```yaml
# VIOLATION in Pod spec
spec:
  hostPID: true    # CIS 5.2.2 violation
  hostIPC: true    # CIS 5.2.3 violation
  hostNetwork: true # CIS 5.2.4 violation

# COMPLIANT: All false (or not set)
spec:
  hostPID: false
  hostIPC: false
  hostNetwork: false
```

**CIS 5.2.6 — Minimize the admission of root containers (Level 2):**
```yaml
# COMPLIANT: runAsNonRoot enforced
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
    runAsGroup: 1000
    fsGroup: 2000
  containers:
  - name: app
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop: ["ALL"]
```

**CIS 5.2.9 — Minimize the admission of containers with added capability (Level 1):**
```yaml
# VIOLATION: Adding dangerous capabilities
securityContext:
  capabilities:
    add: ["SYS_ADMIN", "NET_ADMIN"]  # CIS 5.2.9 CRITICAL violation

# COMPLIANT: Drop all, add only minimum required
securityContext:
  capabilities:
    drop: ["ALL"]
    add: ["NET_BIND_SERVICE"]  # Only if binding to port < 1024
```

**CIS 5.4.1 — Prefer using secrets as files over secrets as environment variables (Level 1):**
```yaml
# VIOLATION: Secret as env var (visible in process list, logs)
env:
- name: DB_PASSWORD
  valueFrom:
    secretKeyRef:
      name: db-secret
      key: password

# COMPLIANT: Secret as mounted file
volumeMounts:
- name: db-credentials
  mountPath: /secrets/db
  readOnly: true
volumes:
- name: db-credentials
  secret:
    secretName: db-secret
```

**CIS 5.7.1 — Create administrative boundaries between resources using namespaces (Level 1):**
```yaml
# VIOLATION: All workloads in default namespace
metadata:
  namespace: default

# COMPLIANT: Dedicated namespaces per application/team
metadata:
  namespace: production-payments
```

**CIS 5.7.3 — Apply SecurityContext to pods and containers (Level 2):**
Full security context template:
```yaml
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
    runAsGroup: 1000
    fsGroup: 2000
    seccompProfile:
      type: RuntimeDefault
  containers:
  - name: app
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop: ["ALL"]
```
