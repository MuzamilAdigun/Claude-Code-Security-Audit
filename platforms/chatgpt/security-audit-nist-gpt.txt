## NIST SP 800-53 Rev 5 Control Family Checks

### AC — Access Control (26 controls in Moderate baseline)

**AC-1 — Access Control Policy and Procedures:**
- Documented access control policy referenced in SECURITY.md or equivalent
- Roles defined with specific permissions documented

**AC-2 — Account Management:**
- Account provisioning and deprovisioning process exists
- Privileged accounts enumerated and reviewed regularly
- Dormant accounts disabled after defined period (30–90 days)
- Service accounts have minimum required permissions

```python
# AC-2 compliant account management
class UserAccount:
    last_login: datetime
    account_status: Enum['active', 'suspended', 'disabled']
    privilege_level: Enum['standard', 'elevated', 'admin']
    review_date: datetime

    def is_dormant(self) -> bool:
        return (datetime.utcnow() - self.last_login).days > 90

    def auto_disable_if_dormant(self):
        if self.is_dormant():
            self.account_status = 'disabled'
            audit_log.warning("Account auto-disabled: dormant",
                              extra={"user_id": self.id, "last_login": self.last_login})
```

**AC-3 — Access Enforcement:**
- Least-privilege access enforced at every layer (API, database, file system)
- Access decisions made server-side, never client-side
- Resource ownership verified before access granted

**AC-4 — Information Flow Enforcement:**
- Data cannot flow from high-security to low-security zones without explicit controls
- API responses filter data based on caller's security clearance/role
- No sensitive data leakage in error responses

**AC-6 — Least Privilege:**
```python
# AC-6 violation: Overly broad database permissions
DB_USER = "root"  # Full database admin — violation

# AC-6 compliant: Read-only service account
DB_READ_USER = "app_read"  # SELECT only on specific tables
DB_WRITE_USER = "app_write"  # INSERT/UPDATE on specific tables only
# No DROP, TRUNCATE, or schema modification permissions
```

**AC-7 — Unsuccessful Logon Attempts:**
- Account lockout after 3–5 failed attempts (Moderate baseline)
- Lockout duration configurable (15–30 minutes minimum)
- Lockout events logged and alerted

**AC-8 — System Use Notification:**
- Warning banner on login page (for government-adjacent systems)
- Terms of service acknowledgment for sensitive systems

**AC-11 — Device Lock / Session Lock:**
- Session timeout after 15 minutes of inactivity
- Automatic logout after maximum session duration

**AC-17 — Remote Access:**
- VPN or zero-trust access required for administrative functions
- Multi-factor authentication required for remote access
- Remote sessions logged and monitored

### AU — Audit and Accountability (12 controls)

**AU-2 — Event Logging:**
Required events to log (Moderate baseline):
- Authentication events (success and failure)
- Authorization failures (access denied)
- Account management events (create, modify, disable, delete)
- Privileged function use
- Changes to audit configuration
- Detected security events

**AU-3 — Content of Audit Records:**
Each log entry must contain:
- Date and time
- Type of event
- Subject identity (user ID, service account)
- Outcome (success/failure)
- Source of event (IP address, hostname)
- Objects affected (resource identifier)

```python
# AU-3 compliant structured logging
import structlog
from datetime import datetime

def audit_event(event_type: str, subject_id: str, outcome: str,
                resource: str = None, source_ip: str = None, **kwargs):
    structlog.get_logger().info(
        event_type,
        timestamp=datetime.utcnow().isoformat() + "Z",
        subject_id=subject_id,
        outcome=outcome,            # 'success' or 'failure'
        resource=resource,
        source_ip=source_ip or request.remote_addr,
        service=os.environ.get('SERVICE_NAME'),
        **kwargs
    )
```

**AU-6 — Audit Record Review, Analysis, and Reporting:**
- Log review process defined (automated alerting + periodic manual review)
- Audit reduction tools or SIEM configured

**AU-9 — Protection of Audit Information:**
- Logs protected from unauthorized modification (append-only, write-once)
- Log integrity verified (cryptographic hashing or SIEM chain of custody)
- Log backup separate from production system

**AU-11 — Audit Record Retention:**
- Minimum 90 days immediately accessible
- Minimum 1 year total retention (NIST 800-53 Moderate)
- Retention configuration visible in logging setup

**AU-12 — Audit Record Generation:**
- All required AU-2 events generating log entries
- Logging cannot be disabled by application users

### CM — Configuration Management (9 controls)

**CM-2 — Baseline Configuration:**
- Infrastructure defined as code (all configuration in version control)
- Baseline configuration documented and enforced
- No undocumented manual changes to production

**CM-3 — Configuration Change Control:**
- All changes via PR with mandatory review
- Security review gate in CI/CD for configuration changes
- Emergency change procedure documented

**CM-6 — Configuration Settings:**
- Security-relevant configuration parameters defined and enforced
- Hardening guides applied (CIS benchmarks for OS, containers, cloud)
- Deviation from baseline requires documented justification

**CM-7 — Least Functionality:**
- Only necessary services, protocols, and ports enabled
- Unnecessary software removed or disabled
- Default accounts and services disabled

**CM-8 — System Component Inventory:**
- Software component inventory maintained (SBOM — Software Bill of Materials)
- All dependencies tracked with versions

### IA — Identification and Authentication (11 controls)

**IA-2 — Identification and Authentication (Organizational Users):**
- Unique IDs for all users — no shared accounts
- MFA required for all privileged access (Moderate baseline)
- MFA required for network access to privileged accounts

**IA-5 — Authenticator Management:**
- Password minimum 12 characters (NIST 800-63B Rev 4 guidance)
- Check against known compromised password lists (HaveIBeenPwned API)
- No password complexity rules that lead to weak predictable patterns
- Password change when compromised (not forced periodic changes per NIST 800-63B)
- Cryptographically strong token generation for session IDs

```python
# IA-5 compliant password policy (aligned with NIST 800-63B)
import requests

def validate_password(password: str, username: str) -> tuple[bool, str]:
    if len(password) < 12:
        return False, "Password must be at least 12 characters"

    if username.lower() in password.lower():
        return False, "Password must not contain your username"

    # Check against breached passwords (HaveIBeenPwned)
    sha1_hash = hashlib.sha1(password.encode()).hexdigest().upper()
    prefix, suffix = sha1_hash[:5], sha1_hash[5:]
    response = requests.get(f"https://api.pwnedpasswords.com/range/{prefix}")
    if suffix in response.text:
        return False, "This password has appeared in a data breach. Choose a different password."

    return True, "OK"
```

**IA-8 — Identification and Authentication (Non-Organizational Users):**
- External users authenticated before accessing protected resources
- Federated identity standards followed (OAuth 2.0, SAML 2.0, OIDC)
- Guest/anonymous access limited to explicitly defined public resources

### SC — System and Communications Protection (44 controls)

**SC-5 — Denial-of-Service Protection:**
- Rate limiting on all public endpoints
- DDoS mitigation configured (CDN, WAF)
- Connection limits configured

**SC-7 — Boundary Protection:**
- Network segmentation between application tiers
- DMZ for public-facing components
- Ingress and egress filtering documented in IaC

**SC-8 — Transmission Confidentiality and Integrity:**
- TLS 1.2+ for all communications (mandatory)
- Certificate validation enforced (no certificate warnings suppressed)
- Sensitive data encrypted in transit

**SC-12 — Cryptographic Key Establishment and Management:**
- Key generation using approved algorithms (FIPS 140-2/3 validated modules for federal)
- Key storage separate from encrypted data
- Key rotation schedule defined and automated

**SC-13 — Cryptographic Protection:**
- FIPS-approved algorithms used (AES-256, SHA-256+, RSA 2048+, ECDSA P-256+)
- No deprecated algorithms (DES, 3DES, RC4, MD5, SHA-1 for signing)

**SC-28 — Protection of Information at Rest:**
- Database encryption at rest
- Backup encryption
- Sensitive files encrypted

**SC-39 — Process Isolation:**
- Processes run as non-root (least privilege)
- Container security contexts configured (runAsNonRoot, readOnlyRootFilesystem)

```yaml
# SC-39 compliant Kubernetes security context
securityContext:
  runAsNonRoot: true
  runAsUser: 1000
  readOnlyRootFilesystem: true
  allowPrivilegeEscalation: false
  capabilities:
    drop: ["ALL"]
```

### SI — System and Information Integrity (17 controls)

**SI-2 — Flaw Remediation:**
- Automated vulnerability scanning in CI/CD
- Patch SLA defined and enforced (Critical: 24h, High: 7d, Medium: 30d)
- Vulnerability tracking in issue system

**SI-3 — Malware Protection:**
- File upload scanning
- Container image scanning for malware signatures
- Endpoint protection on build agents

**SI-4 — System Monitoring:**
- Intrusion detection system (IDS) configured
- Anomaly detection for unusual access patterns
- Real-time alerting for security events

**SI-10 — Information Input Validation:**
- All user inputs validated against defined schemas
- Input length, type, format, and range enforced
- Whitelist (allowlist) validation preferred over blacklist

```python
# SI-10 compliant input validation
from pydantic import BaseModel, EmailStr, constr, validator

class UserRegistration(BaseModel):
    email: EmailStr
    username: constr(min_length=3, max_length=50, regex='^[a-zA-Z0-9_-]+$')
    password: constr(min_length=12, max_length=128)

    @validator('username')
    def username_not_reserved(cls, v):
        reserved = ['admin', 'root', 'system', 'api', 'null']
        if v.lower() in reserved:
            raise ValueError('Username is reserved')
        return v
```

**SI-12 — Information Management and Retention:**
- Data retention schedules defined and enforced programmatically
- Automated deletion at end of retention period
