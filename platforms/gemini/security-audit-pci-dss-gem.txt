## PCI-DSS Scope Assessment

**First, I determine:**

1. Does the application store, process, or transmit Primary Account Numbers (PAN)?
2. Does it handle Sensitive Authentication Data (SAD): CVV2/CVC, PIN, full magnetic stripe data?
3. Does it use a hosted payment page (iframe, redirect) so CHD never touches your servers?
4. Does it use a payment SDK (Stripe.js, Braintree Drop-in UI) that tokenizes before data reaches your servers?

**Scope reduction scenarios:**
- **Stripe.js / Stripe Elements:** PAN never hits your servers → SAQ A eligible (minimal requirements)
- **Stripe Checkout (redirect):** Fully out of scope for CHD → SAQ A-EP
- **Direct API with card data:** Full PCI-DSS scope → SAQ D or ROC required
- **Stored PANs in database:** Requirement 3 becomes critical

## PCI-DSS v4.0 Requirements Checks

### Requirement 1 — Install and Maintain Network Security Controls

**What I analyze:**
- Network segmentation between cardholder data environment (CDE) and other systems
- Firewall rules defined in IaC (only necessary ports open)
- DMZ configuration for public-facing components
- Inbound/outbound traffic rules documented and enforced
- Default deny-all rule with explicit allow list

**Gap example:**
```terraform
# Overly permissive security group — Req 1 violation
resource "aws_security_group_rule" "allow_all" {
  type        = "ingress"
  from_port   = 0
  to_port     = 65535
  protocol    = "tcp"
  cidr_blocks = ["0.0.0.0/0"]  # All traffic from internet — never acceptable in CDE
}
```

**Compliant example:**
```terraform
resource "aws_security_group_rule" "allow_https" {
  type        = "ingress"
  from_port   = 443
  to_port     = 443
  protocol    = "tcp"
  cidr_blocks = ["0.0.0.0/0"]  # HTTPS only from internet
}
resource "aws_security_group_rule" "allow_db_from_app" {
  type                     = "ingress"
  from_port                = 5432
  to_port                  = 5432
  protocol                 = "tcp"
  source_security_group_id = aws_security_group.app.id  # Only from app tier
}
```

### Requirement 3 — Protect Stored Account Data (CRITICAL)

**What I analyze:**
- PANs not stored unless absolutely necessary (data retention minimization)
- If stored: PANs masked (show only last 4 digits) in all displays and logs
- If stored: PANs encrypted using AES-256 or equivalent
- SAD (CVV2, CVC, PIN block) NEVER stored after authorization — even encrypted
- Key management: encryption keys stored separately from encrypted data
- No PANs in logs, debugging output, error messages, or non-payment databases
- Tokenization used instead of storing PANs where possible

**Critical violations:**
```python
# CRITICAL PCI VIOLATION — Never store CVV/CVC
db.execute("INSERT INTO payments (pan, cvv, expiry) VALUES (?, ?, ?)",
           pan, cvv, expiry)  # CVV must NEVER be stored

# CRITICAL — Logging full PAN
logger.info(f"Processing card: {card_number}")  # Must never log PAN

# CRITICAL — Unencrypted PAN in database
db.execute("INSERT INTO transactions (pan, amount) VALUES (?, ?)", pan, amount)
```

**Compliant approaches:**
```python
# Store only token — never the PAN
stripe_token = stripe.PaymentMethod.create(...)
db.execute("INSERT INTO transactions (stripe_token, amount) VALUES (?, ?)",
           stripe_token.id, amount)

# Log only masked PAN
def mask_pan(pan):
    return '*' * (len(pan) - 4) + pan[-4:]

logger.info(f"Processing card ending in {pan[-4:]}")

# Never store CVV — remove from object immediately after authorization
del payment_data['cvv']
del payment_data['cvc']
```

### Requirement 4 — Protect Cardholder Data with Strong Cryptography During Transmission

**What I analyze:**
- TLS 1.2+ enforced for all CHD transmission (no TLS 1.0, 1.1, SSL)
- HSTS header with minimum 1-year max-age on all payment pages
- TLS certificate valid and not expired
- Weak cipher suites disabled
- No CHD transmitted over unencrypted channels
- Certificate pinning for payment-critical API calls (mobile apps)

**Gap example:**
```python
# Accepting old TLS versions — Req 4 violation
ssl_context = ssl.SSLContext(ssl.PROTOCOL_TLS)
ssl_context.options |= ssl.OP_NO_SSLv2
ssl_context.options |= ssl.OP_NO_SSLv3
# Missing: OP_NO_TLSv1, OP_NO_TLSv1_1
```

**Compliant:**
```python
ssl_context = ssl.SSLContext(ssl.PROTOCOL_TLS_SERVER)
ssl_context.minimum_version = ssl.TLSVersion.TLSv1_2
ssl_context.set_ciphers('ECDHE+AESGCM:ECDHE+CHACHA20:!aNULL:!MD5:!DSS')
```

### Requirement 8 — Identify Users and Authenticate Access

**What I analyze:**
- Unique IDs for all users (no shared accounts)
- MFA required for all non-consumer accounts with CDE access
- Strong password policy enforced (minimum 12 characters, complexity requirements per v4.0)
- Session tokens expire after 15 minutes of inactivity
- Passwords hashed with strong, salted algorithms (bcrypt, Argon2, PBKDF2)
- Account lockout after maximum 10 failed attempts (v4.0 requirement)

```python
# PCI-DSS v4.0 Req 8 compliant authentication
password_policy = {
    'min_length': 12,
    'require_upper': True,
    'require_lower': True,
    'require_digit': True,
    'require_special': True,
    'max_age_days': 90,      # Force change every 90 days
    'history_count': 4,      # Cannot reuse last 4 passwords
    'lockout_attempts': 10,  # Lock after 10 failures
    'lockout_duration': 30,  # 30-minute lockout
}

# Session timeout (15 minutes inactivity)
SESSION_COOKIE_AGE = 900  # 15 minutes
SESSION_SAVE_EVERY_REQUEST = True
```

### Requirement 10 — Log and Monitor All Access to System Components and Cardholder Data

**What I analyze:**
- All access to CHD logged (read, create, update, delete)
- Authentication events logged (success and failure)
- Privilege escalation logged
- Log entries include: timestamp, user ID, event type, data affected, originating system, success/failure
- Logs retained for minimum 12 months (3 months immediately available)
- Log integrity protection (cannot be modified by regular users)
- Log review process with alerting

```python
# Compliant PCI logging for CHD access
def log_chd_access(user_id, action, record_id, success, ip_address):
    audit_logger.info(json.dumps({
        "timestamp": datetime.utcnow().isoformat() + "Z",
        "user_id": user_id,
        "event_type": "chd_access",
        "action": action,         # read/create/update/delete
        "record_id": record_id,   # Token or last-4 reference, NOT full PAN
        "success": success,
        "source_ip": ip_address,
        "service": "payment-api"
    }))
```
